章节3.1.2

1.数据完整性约束
数据完整性约束指的是为了防止不合规范的数据进入数据库，在用户对数据进行插入、修改、删除等操作时，DBNS自动按照一定的约束条件对数据进行监测，使不符合规范的数据不能进入数据库以确保数据中存储的数据正确、有效、相容。

2.数据完整性约束是一组完整性规则的集合，规则包含：

  • 实体完整性
    • 实体完整性：规定表的每一行在表中是惟一的实体。实体完整性是指数据表中的所有行都是惟一的确定的，所有记录都是可以区分的（满足2NF）。实体完整性规则规定了表中的主键值惟一，所有主要属性都非空。
  • 参照完整性
    • 参照完整性：是指两个表的主关键字和外关键字的数据应一致，当一个表中的某列数据（外键）依赖并引用另一个表中的某列数据时，这两个表之间的相关数据必須保持一致性。
    • 例：XS（学号，姓名，性别……
    • KC（课程号，课程名，学分）
    • CJ（学号，课程号，成绩）
  • 域完整性
    • 域完整性：域完整性是指表中每列的数据具有正确的数据类型、格式和有效的取值范围，保证数据的正确性。
  • 用户定义完整性
    • 用户定义的完整性：用户自定义完整性是针对某一具体关系数据库的约束条件，它反映所涉及的数据必须满足的语义要求。主要包括非空约束、唯一约束、检查约柬、主键约束、外键约束.

3.常见的数据完整性约束

  • not null
    • not null:(非空)约束:只用于定义列约束
  • unique
    • unique：用于指明列上的取值必须唯一
  • primary key
    • primary key：用于定义基本表的主键，起唯一标识符作用，其值不能为null，也不能重复，以此来保证实体的完整性。
  • foreign key
    • foreign key：定义了一个表中数据与另一个表中的数据的联系。
  • check
    • check：用来检查字段值所允许的范围。DBMS每当执行delete、insert或update数据操作语句时，都对这个约束过滤.如果为true，则执行。否则，取消执行并提示错误。
  • default

mysql网课记录（4）

发表于 2020-03-31 | 分类于 网课记录

章节3.1

1.sql语言
结构化查询语言简称SQL语言，是一种特殊目的的编程语言，是一种数据库查询和程序设计语言，用于查询数据及查询，更新和管理关系数据库系统。

2.sql语言功能上分三部分

  • DDL:数据定义语句
  • DML:数据操纵语言
  • DCL：数据控制语句

2.1DDL
数据定义语言（DDL）。用于执行数据库的任务，对数据库及数据库中的各种对象进行创建、删除、修改等操作。如前所述，数据库对象主要包括：表默认约束、规则、视图、触发器、存储过程等。DDL包括的主要语句及功能如表所示。

语句 功能 说明
create 创建数据库及数据库对象 不同数据库对象，其create语句的语法形式不同
alter 对数据库或数据库对象进行修改 不同数据库对象，其alter语句的语法形式不同
drop 删除数据库或数据库对象 不同数据库对象，其drop语句的语法形式不同

2.2DML
数据操纵语言（DML）。用于操纵数据库中各种对象，检索和修改数据。DML包括的主要语句及功能如表所示。

语句 功能 说明
SELECI 从表或视图中检索数据 是使用最频繁的SQL语句之一
INSERT 数据插入到表或视图中
UPDATE 修改表或视图中的数据 既可修改表或视图的，也可修改一组或全部数据
DELETE 从表或视图中删除数据 可根据条件删除指定的数据

2.3DCL
数据控制语言（DCL）。用于安全管理，确定那些用户可以查看或修改数据库中的数据，DCL包括的主要语句及功能如表所示。

语句 功能 说明
grant 授予权限 可把语句许可或对象许可的权限授予其他用户和角色
revoke 收回权限 收回权限与grant的功能相反，但不影响该用户或角色从其他角色中作为成员继承许可权限

2.4
My SQL增加的语言元素。这部分不是SQL标准所包含的内容，而是为了用户编程的方便增加的语言元素。这些语言元素包括常量、变量、运算符、函数、流程控制语句和注解等。

3.1
mysql的数据类型包括：

  • 数值类型
    • 整数类型
    • 浮点类型
    • 定点数型
  • 字符串类型
  • 日期时间类型

3.1.1 整数类型
整数类型的划分是根据存储空间的大小和表示范围的大小来划分的，具体如下表所示

数据类型 存储空间（Bytes） 表示范围
TINY INT 1 -128~127
SMALL INT 2 -32768~32767
MEDIUM INT 3 -8388608~8388607
INT 4 -2147483648~2147483647
BIG INT 8 ±9.22×10的18次方

3.1.2 浮点数类型
浮点数类型是用来表示实数的一种方法，根据位数和精度，浮点数类型又分为单精（ FLOAT）浮点数和双精浮点数（ DOUBLE），具体如下表所示。

数据类型 存储空间 精度
FLOAT（M, D） 4 8位精度，M表示数字的总位数，D表示小数点后面数字的位数
DOUBLE（M, D） 8 16位精度，M表示数字的总位数，D表示小数点后面数字的位数

3.1.2字符串类型

  • char：固定长度字符串
  • varchar：可变长度字符串
  • Binary：无字符集的字符串
  • Varbinary：可变长无字符集的字符串
  • Blob和text：以对象类型保存，足够大的数据
  • Enum：枚举类型，从值集合中选取单个值
  • Sct：集合对象，从值集合中迭取多个值

#章节2.3

1.添加用户
create user “用户名”@”主机名(本地一般是localhost）” identified by “密码”;

2.授予用户权限
grant privileges on 数据库名.表名 to 用户
其中 privileges 可以包含
create（创建） alter（改变） drop（删除）： 库，表结构操作
select（选择） insert(插入） delete（删除） update（更新）：数据操作
references:外键权限
index:索引

演示
为用户xm授予STUMARKDB数据库中所有表的select、insert、create及alter的权利。

3.删除用户的权限
revoke privileges(要回收的权限) on 数据库名.表名 from 用户

演示
删除用户xm的select权限

4.修改用户名
rename user ‘老用户名‘@’localhost’ to ‘新用户名‘@’localhost’;

5.修改用户密码
set password for’用户名‘@’localhost’=password(‘密码’);

6.删除用户
drop user ‘用户名‘@’localhost’;
