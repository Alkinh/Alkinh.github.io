#4.2 数据库分区

4.2.1 分区概述
表分区就是将一个大表按照mysq提供的几种方式，分成几个小表.
如果一个表的数据量太大，不仅查找数据的效率低，而且难以找到一块集中的存储空间来存放。通过分区可以在物理层面将一个表分割成许多个小块进行存储，这样我们在査找数据时，就不需要进行全表查找了，只需要知道这条数据所在的块号，然后根据块号直接定位相应的块进行查找即可。分区也可以将数据分配到不同的磁盘中，以解决一块磁盘放不下整个表的问题。

4.2.2 MySQL分区的优点和不足

优点 不足
可以存储更多的数据 一个表最多只能有1024个分区
优化查询 在 MySQL5.1之后才支持分区
对于不需要的数据可通过删除区块快速删除 MySQL5.1只支持整型表达式进行分区， MySQL5.5后可以支持其他类型表达式进 行分区，也不是所有类型
跨多个磁盘来分散数据查询，可获得更大的查询吞吐量 分区表无法使用外键约束

4.2.3 分区类型

  1. RANGE分区
    RANGE分区是基于一个给定连续区间范围，把记录分配到不同的分区。
  2. LIST分区
    LIST分区类似与 RANGE分区，区别在于LIST分区基于列值匹配一个离散值集合中的某个值来进行分区。
  3. HASH分区
    HAST分区是基于用户定义的表达式返回值来选择分区，该表达式对要插入到表中的列值进行哈希计算（Hash计算）。
  4. KEY分区
    KEY分区类似与HASH分区，但是HASH分区允许用户使用自定义表达式，而KEY分区不允许，它需要使用 My SQL提供的HASH函数，同时HASH分区只支持整数类型，而KEY分区支持除BLOB和TEXT类型外的其他类型

4.1 数据库索引

1.索引及作用
1.1索引
索引是根据表巾一列或若干列按照一定顺序建立的列值与记录行之间的对应关系表。在列上创建了索引之后，查找数据时可以自接根据该列上的索引找到对应行的位置，从而快速地找到数据。
例如
如果用户创建了学生表中学号列的索引， MySQL将在索引中排序学号列，对于索引中的每一项， MySQL在内部为它保行一个数据文件中实际记录所在位智的“指针”。因此，如果要查找学号为“2019001”的学生信息， MySQL能在学号列的索引中找到“2019001”的值，然后直接转到数据文件中相应的行，准确地返回该行的数据。

1.2索引作用
在数据库系统中建立索引主要有以下作用

  • 快速读取数据
  • 保证数据记录的唯一性
  • 实现表与表之间的参照完整性
  • 在使用 GROUP BY、 ORDER BY子句进行数据检索吋，利用索引可减少排序和分组的时间。

1.3MySQL索引
在 MySQL5.6中，所有的 MySQL列类型都能被索引，但要注意以下几点只有当表类型为 MyISAM、 InnoDB或BDB时，才可以向有NULL、BLOB或TEXT列中添加索引。
一个表最多可有16个索引。最大索引长度足256个字节。
对于CHAR和VARCHAR列，可以索引列的前缀。这样索引的速度吏快并且比索引整个列需要较少的磁盘空间。
MySQL能在多个列上创建索引。索引可以由最多15个列组成（在CHAR和VARCHAR列上，也可以使用列的前缀作为索引的部分）。

2.索引的分类
2.1 BTREE索引目前大部分 MySQL索引都是以B-树（ BTREE）方式存储的，索引类型分成下列几个。

  • （1）普通索引（ INDEX） 这是最基本的索引类型，它没有唯一性之类的限制。创建普通索引的关键字是INDEX。
  • （2）唯一性索引（ UNIQUE) 这种索引和前面的普通索引基本相同，但有一个区别：索引列的所有值都只能出现一次，即必须是唯一的。创建唯一性索引的关键字是 UNIQUE
  • （3）主键（ PRIMARY KEY）主键是种唯性索引，它必须指定为“ PRIMARY KEY”。主键般在创建表的时候指定，也可以通过修改表的方式加入主键。但是每个表只能有一个主键
  • （4）全文索引（ FULLTEXT）MySQL支持全文检索和全文索引。在 My SQL中，个文索引的索引类型为FULLTEXT。

2.2 MySQL索引创建
(1)CREATE|NDEX语句创建使用 CREATE|NDEX语句可以在一个已有表上创建索引，一个表可以创建多个索引。

语法格式
CREATE[UNIQUE| FULLTEXT| SPATIAL]NDEX索引名
[索引类型]
ON表名（索引列名…）
[索引类型]
索引列名=：
列名[(长度)][ASC|DESC]

说明
UNIQUE| FULLTEXT| SPATIAL：∪NQUE表示创建的足唯一性索引； FULLTEXT表示创建全文索引； SPATIAL表示为空间索引，可以用来索引几何数据类型的列（本书不讨论这种索引）。
索引名：索引在一个表中名称必须是唯一的
索引类型：MySQL支持的索引类型有 BTREE和HASH。
索引列名：创建索引的列名后的长度表示该列前面创建索引字符个数。这可使索引文件大大减小，从而节省磁盘空间

ON条件
|USING （column_list)

左外连接（ LEFT OUTER JOIN）：结果表中除了匹配行外，还包括左表有的但右表中不匹配的行，对于这样的行，从右表被选择的字段设置为NULL。

3.6.8分组查询（聚合函数）

1.聚合函数
聚合函数常常用于对一组值进行计算，然后返回单个值。除 COUNT函数外，聚合函数都会忽略空值。聚合函数通常与GROUP BY子句一起使用。如果包含聚合函数的 SELECT语句中有 GROUP BY子句，则会返回多行记录，如果没有，则 SELECT语句只产生一行记录作为结果。

1.1常用聚合函数表

函数名 说明
COUNT 求组中项数，返回int类型整数
MAX 求最大值
MIN 求最小值
SUM 返回表达式中所有值的和
AVG 求组中值的平均值

1.2 COUNT函数
聚合函数中最经常使用的是 COUNT函数，用于统计组中满是条件的行数或总行数，返回 SELECT语句检索到的行中非NULL值的数目，若找不到匹配的行，则返回0。
语法格式：
COUNT（{[ALL|DISTINCT]表达式}|*)

1.2.1 示例
(1) 求学生的总人数

(2)求男生的人数

(3)求填了性别的学生的总人数

(4)求有学生的班级总数

1.3 MAX和MIN
MAX和MIN分别用于求表达式中所有值项的最大值与最小值，语法格式为：
MAX/MIN（[ALL|DISTINCT]表达式）
其中，表达式可以是常量、字段、函数或表达式，其数据类型可以是数字字符和时间日期类型。

1.3.1 示例
(1)求c001课程的最高分和最低分

1.4 SUM函数和AVG函数
SUM和AVG分别用于求表达式中所有值项的总和与平均值，语法格式为：
SUM/AVG([ALL| DISTINCT]表达式)
其中，表达式可以是常量、字段、函数或表达式，其数据类型只能是数值型数据。

关键字ASC表示升序排列，DESC表示降序排列，系统默认值为ASC

语法格式如下：
SELECT UNION…
UNION [ALL|DISTINCT] SELECT …
[UNION[ALL|DISTINCT] SELECT …]
UNION(UNION DISTINCT)和 UNION ALLI的主要区别：UNION ALL用于将结果集直接合并在一起，而UNON（ UNION DISTINCT）用于将UNION ALL后的结果进行一次不重复处理，以删除重复结果记录。

  1. 联合查询就是把多个查询结果进行组合显示
  2. 多个查询结果集返回的字段数量必须一样。
  3. 联合查询结果集的字段名来源于第一查询语句的字段名
  4. ALL：保留所有记录
    DISTINCT:去除重复记录

3.范围比较
用于范围比较的关键字有两个：BETWEEN和IN。
当要查询的条件是某个值的范围时，可以使用 BETWEEN关键字。

表达式[NOT] BETWEEN 值1 AND 值2
当不使用NOT时，若表达式的值在值1与2之间（包括这两个值），则返回TRUE，否则返回 FALSE；使用NOT时，返回值刚好相反。
使用IN关键字可以指定一个值表，值表中列出所有可能的值，当与值表中的任一个匹配时，即返回TRUE，否则返回 FALSE。使用N关键字指定值表的格式为：
表达式IN（值1，值2，值3，…，值n）

4.空值查询
当需要判定一个表达式的值是否为空值时，使用 IS NULL关键字，格式为：
表达式IS [NOT] NULL
当不使用NOT时，若表达式的值为空值，返回TRUE，否则返回FALSE；当使用N○T时，结果刚好相反。
