3.6.9 连接查询

1.连接查询
在 SELECT语句中，如果FROM子句引用了多个表源或视图，则可以使用JOIN关键字指示指定的连接操作在指定的表源和视图之间执行。
语法格式：
join_table:
table_reference[INNER|CROSS]JOIN table_factor [join_condition]
| table_reference{LEFT|RIGHT}[OUTER]JOIN table_reference join_condition
其中，join_condotion:
ON条件
|USING （column_list)

1.1例

(1)查找选修了c001课程且成绩在80分以上的学生姓名及成绩。

(2)查询出每个班级每门课程的平均分，显示出班级名称、课程名称及平均分，按平均分查询。

2.外连接
制定了OURTER关键字的连接为外连接。
外连接包括：
左外连接（ LEFT OUTER JOIN）：结果表中除了匹配行外，还包括左表有的但右表中不匹配的行，对于这样的行，从右表被选择的字段设置为NULL。
右外连接（RIGHT OUTER JOIN）：结果表中除了匹配行外，还包括右表有的但左表中不匹配的行，对于这样的行，从左表被选择的字段设置为NULL。

2.1例
(1)查找所有学生情况及他们选课情况，若学生未选修任何课，也要包括其情况。

3.6.8分组查询（聚合函数）

1.聚合函数
聚合函数常常用于对一组值进行计算，然后返回单个值。除 COUNT函数外，聚合函数都会忽略空值。聚合函数通常与GROUP BY子句一起使用。如果包含聚合函数的 SELECT语句中有 GROUP BY子句，则会返回多行记录，如果没有，则 SELECT语句只产生一行记录作为结果。

1.1常用聚合函数表

函数名 说明
COUNT 求组中项数，返回int类型整数
MAX 求最大值
MIN 求最小值
SUM 返回表达式中所有值的和
AVG 求组中值的平均值

1.2 COUNT函数
聚合函数中最经常使用的是 COUNT函数，用于统计组中满是条件的行数或总行数，返回 SELECT语句检索到的行中非NULL值的数目，若找不到匹配的行，则返回0。
语法格式：
COUNT（{[ALL|DISTINCT]表达式}|*)

1.2.1 示例
(1) 求学生的总人数

(2)求男生的人数

(3)求填了性别的学生的总人数

(4)求有学生的班级总数

1.3 MAX和MIN
MAX和MIN分别用于求表达式中所有值项的最大值与最小值，语法格式为：
MAX/MIN（[ALL|DISTINCT]表达式）
其中，表达式可以是常量、字段、函数或表达式，其数据类型可以是数字字符和时间日期类型。

1.3.1 示例
(1)求c001课程的最高分和最低分

1.4 SUM函数和AVG函数
SUM和AVG分别用于求表达式中所有值项的总和与平均值，语法格式为：
SUM/AVG([ALL| DISTINCT]表达式)
其中，表达式可以是常量、字段、函数或表达式，其数据类型只能是数值型数据。

1.4.1 示例
(1) 求c001课程的最高分、最低分、选修人数、总分和平均分。

3.6.7 限制查询

1.限制查询
LIMT子句，主要用于限制 SELECT语句返回的行数。
语法格式：
LIMIT{[offest,]row_count}
说明：语法格式中的 offset和row_ count都必须是非负的整数， offset指定返回的第一行的偏移量， row_count是返回的行数。
限制查询的功能：
1.LIMIT限制返回的数量可以有效的减少服务器的压力和传输压力。
2.利用LIMIT来实现分页功能

2.示例
(1)查询出c001课程前三名的成绩

(2)查询出c001课程第四名到第六名的成绩

3.6.6 排序查询

1.排序查询
在一条 SELECT语句中，如果不使用ORDER BY子句，结果中行的顺序是不可预料的。使用 ORDER BY子句后可以保证结果中的行按一定顺序排列。
语法格式
ORDER BY{字段名|表达式|position}[ASC|DESC],…
关键字ASC表示升序排列，DESC表示降序排列，系统默认值为ASC

2.示例
(1)查询tb_student表中的sno,sname,sage,classname,按姓名大小排列。

(2)将19软件4班的学生信息按入学日期先后排序

(3)将tb_mark表中的记录按cno正序及成绩倒序(即同一门课程成绩按照倒序排）排序。

3.6.5 模糊查询

1.模糊查询
LIKE运算符用于指出一个字符串是否与指定的字符串相匹配，其运算对象可以是char、 varchar、text、 datetime等类型的数据，返回逻辑值TRUE或FALSE。LIKE谓词表达式的格式为：
表达式[NOT]LIKE匹配表达式[ESCAPE’escape_character’]
使用LIKE进行模式匹配时，常使用特殊符号_和%，可进行模糊查询。
“%”代表0个或多个字符，“_”代表单个字符。

2.示例
(1)查询tb_student表中，姓王的学生信息。

(2)查询tb_student表中，姓名包含王的学生信息。

(3）查询tb_student表中，姓王且只有一个字的学生信息。

(4)查询tb_student表中，姓王且名字里只有两个字的学生信息。

(5)查询tb_student表中,sno倒数第二个数字为1的学生sno、sname、及classname。

(6)查询tb_course表中，cname包含”_”的课程信息。

3.6.4不重复查询

1.不重复查询
对表只选择其某些字段时，可能会出现重复行。例如，若对 stumarkdb数据库的 tb_student表只选择班级，会出现多行重复的情况。可以使用DISTINCT关键字消除结果集中的重复行，格式是：
SELECT DISTINCT 字段名 …

1.1知识点
(1)ALL：默认，保留全部记录
(2)DISTINCT:去除重复记录

示例
在tb_student表中查询出那些班级有学生

3.6.3 联合查询

1.联合查询
使用UNON关键字，把来自许多 SELECT语句的结果集合到一个结果集合中。
语法格式如下：
SELECT UNION…
UNION [ALL|DISTINCT] SELECT …
[UNION[ALL|DISTINCT] SELECT …]
UNION(UNION DISTINCT)和 UNION ALLI的主要区别：UNION ALL用于将结果集直接合并在一起，而UNON（ UNION DISTINCT）用于将UNION ALL后的结果进行一次不重复处理，以删除重复结果记录。

(1)知识点

  1. 联合查询就是把多个查询结果进行组合显示
  2. 多个查询结果集返回的字段数量必须一样。
  3. 联合查询结果集的字段名来源于第一查询语句的字段名
  4. ALL：保留所有记录
    DISTINCT:去除重复记录

1.1示例
(1)查询出学生表和离校学生表中所有学生的sname，ssex，tel，edate，分别给出别名姓名，性别，电话号码，入学日期。

(2)查询出学生表和离校学生表中所有女生的信息；//联合查询中的条件 要加在所有查询语句后面

3.6.2 条件查询

1.条件查询
在大多数情况下，用户并不需要查询表内所有记录，而是需要根据特定条件来查询部分数据，此时，可以用 WHERE子句实现条件查询。条件查询的语法格式如下：
select 字段名 from 表名 where 条件；
条件=：[not]<判断运算>|<判断运算>{AND|OR}<判断运算>

2.比较运算
条件的判断常常使用比较运算符，常用比较运算符如下表。

比较运算符 说明
= 等于，检测两个值是否相等，如果相等则返回true
<>,!= 不等于，检测两个值是否不相等，如果相等则返回 false
> 大于，检测左边的值是否大于右边的值，如果大于则返回true
< 小于，检测左边的值是否小于右边的值，如果大于则返回 false
>= 大于或等于，检测左边的值是否大于或等于右边的值，如果大于等于则返回true
<= 小于或等于，检测左边的值是否小于或等于右边的值，如果大于等于则返回 false

3.范围比较
用于范围比较的关键字有两个：BETWEEN和IN。
当要查询的条件是某个值的范围时，可以使用 BETWEEN关键字。
BETWEEN关键字指出查询范围，格式为：
表达式[NOT] BETWEEN 值1 AND 值2
当不使用NOT时，若表达式的值在值1与2之间（包括这两个值），则返回TRUE，否则返回 FALSE；使用NOT时，返回值刚好相反。
使用IN关键字可以指定一个值表，值表中列出所有可能的值，当与值表中的任一个匹配时，即返回TRUE，否则返回 FALSE。使用N关键字指定值表的格式为：
表达式IN（值1，值2，值3，…，值n）

表达式IS [NOT] NULL
当不使用NOT时，若表达式的值为空值，返回TRUE，否则返回FALSE；当使用N○T时，结果刚好相反。
